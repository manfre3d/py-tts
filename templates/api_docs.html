API Reference

py-tts exposes a lightweight REST API for programmatic text-to-speech conversion.

Base URL
https://py-tts.onrender.com
GET /api/voices

Returns the list of available TTS voices.

Example
curl {{ request.host_url }}api/voices
Response
{{ '{"voices": ' + voices | tojson + '}' }}
GET /api/status

Reports whether AI text cleanup is available (requires OPENAI_API_KEY).

Example
curl {{ request.host_url }}api/status
Response
{"ai_available": true}
POST /api/convert

Converts a PDF, DOCX, or TXT file to MP3. Accepts multipart/form-data. Returns the MP3 file directly as audio/mpeg.

Parameters
Field Type Required Description
file File Yes PDF, DOCX, or TXT file (max 16 MB)
voice String No TTS voice name (default: Matthew). See /api/voices.
Example
curl -X POST {{ request.host_url }}api/convert \
  -F "file=@document.pdf" \
  -F "voice=Joanna" \
  --output audio.mp3
Error responses
400  {"error": "Unsupported file type. Accepted: pdf, txt, docx"}
400  {"error": "Unknown voice. Supported voices: [...]"}
500  {"error": "<message>"}
Notes
  • The /api/convert endpoint is synchronous. Large documents (>50 pages) may take 30–60 seconds.
  • For real-time progress updates, use the web interface which streams progress via Server-Sent Events.
  • No authentication is required. Rate limiting is enforced by the underlying TTS API.
